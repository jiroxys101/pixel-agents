# Data Models — PixelOffice

_Generated: 2026-03-12 | Scan: Exhaustive_

PixelOffice does not use a database. All persistent data is stored as JSON files. This document covers the schema for all persisted and runtime data structures.

---

## Persisted Data

### 1. Layout File — `~/.pixel-agents/layout.json`

The primary user-level persistence store for office layout. Shared across all VS Code windows.

```typescript
interface OfficeLayout {
  version: 1;                          // Schema version (only 1 exists)
  cols: number;                        // Grid width (default: 20, max: 64)
  rows: number;                        // Grid height (default: 11, max: 64)
  tiles: TileType[];                   // Flat array, length = cols × rows, row-major order
  furniture: PlacedFurniture[];        // All placed furniture items
  tileColors?: (FloorColor | null)[];  // Per-tile colorization; length = cols × rows
                                       // null = no color override
}

// Tile index: col + row * cols
const TileType = {
  VOID:  0,  // Transparent, non-walkable, no furniture placement
  FLOOR: 1,  // Walkable, furniture-placeable
  WALL:  2,  // Non-walkable, wall-placeable furniture only
} as const;

interface PlacedFurniture {
  uid: string;         // Unique identifier (crypto.randomUUID())
  type: string;        // References FurnitureCatalogEntry.id
  col: number;         // Left-most column of furniture footprint
  row: number;         // Top-most row (may be negative for wall-placeable items)
  color?: FloorColor;  // Per-item color override (undefined = use default)
}

interface FloorColor {
  h: number;            // Hue (0–360 degrees)
  s: number;            // Saturation (0–100%)
  b: number;            // Brightness offset (-100 to 100)
  c: number;            // Contrast offset (-100 to 100)
  colorize?: boolean;   // true = Photoshop Colorize mode (grayscale → fixed HSL)
                        // false/undefined = Adjust mode (shift existing HSL)
}
```

**File I/O:**
- Write: `writeLayoutToFile()` — writes to `layout.json.tmp`, then renames to `layout.json` (atomic)
- Read: `readLayoutFromFile()` — JSON.parse with error recovery
- Watch: `watchLayoutFile()` — `fs.watch` + 2s polling interval

**Migration:** On first load, checks workspace state key `pixel-agents.layout`; migrates to file if found. Tile type values may be remapped for legacy layouts.

### 4. Furniture Catalog — `dist/assets/furniture-catalog.json`

Static asset bundled with the extension. Generated by `scripts/5-export-assets.ts`.

```typescript
interface FurnitureCatalogEntry {
  id: string;                   // Unique type key, e.g. "DESK_FRONT", "CHAIR_BACK"
  name: string;                 // Internal name
  label: string;                // Display label in editor palette
  category: string;             // "desks" | "chairs" | "storage" | "electronics"
                                // | "decor" | "wall" | "misc"
  footprint: [number, number];  // [cols, rows] bounding box
  isDesk?: boolean;             // Surface items can be placed on this
  canPlaceOnWalls?: boolean;    // Must be placed on WALL tiles (bottom row)
  canPlaceOnSurfaces?: boolean; // Can overlap desk tiles
  groupId?: string;             // Shared key for rotation group OR state group
  orientation?: string;         // "front" | "back" | "left" | "right"
  state?: string;               // "on" | "off" (for toggleable electronics)
  backgroundTiles?: number;     // Top N rows allow character walk-through + overlap
}

// Full catalog: FurnitureCatalogEntry[]
```

**Naming convention:** `{BASE}[_{ORIENTATION}][_{STATE}]`
- Examples: `DESK_FRONT`, `MONITOR_FRONT_OFF`, `MONITOR_FRONT_ON`, `CRT_MONITOR_BACK`
- Rotation group: all entries sharing `groupId` cycle via R key in editor
- State group: entries sharing `groupId` + `orientation` with different `state` values toggle via T key

## Runtime Data Structures (In-Memory)

These are not persisted; they are rebuilt from the layout + JSONL each session.

### Character

```typescript
interface Character {
  id: number;               // Positive for real agents (1, 2, 3...)
                            // Negative for subagents (-1, -2, -3...)
  agentId: number;          // Same as id for real agents; parent's id for subagents
  palette: number;          // 0-5
  hueShift: number;         // Hue rotation degrees (0 for first 6 agents)
  seatId: string | null;    // e.g. "abc123" or "abc123:1" (multi-tile chair)

  x: number;                // Canvas pixel position (center of tile)
  y: number;
  state: 'IDLE' | 'WALK' | 'TYPE';
  direction: 'DOWN' | 'UP' | 'LEFT' | 'RIGHT';
  frame: number;            // 0-6 (walk1-3, type1-2, read1-2)
  frameTimer: number;       // Time in current frame (seconds)
  animationType: 'type' | 'read'; // Determines frame pair used

  path: { col: number; row: number }[] | null;  // Current BFS path
  moveProgress: number;     // 0-1 within current tile move

  // Wander AI (IDLE state)
  wanderTarget: { col: number; row: number } | null;
  wanderMovesLeft: number;
  wanderPauseTimer: number;
  seatTimer: number;        // Countdown before returning to seat

  // Matrix effect
  matrixEffect: 'spawn' | 'despawn' | null;
  matrixEffectTimer: number;

  // Subagent
  isSubagent: boolean;
  parentAgentId?: number;

  // Bubble
  bubble: {
    type: 'permission' | 'waiting' | null;
    fadeTimer?: number;     // For waiting bubble auto-fade (2s)
  };

  isSelected: boolean;      // White outline rendering
  isHovered: boolean;
}
```

## JSONL Transcript Format

Claude Code writes transcripts to `~/.claude/projects/<hash>/<session-id>.jsonl`. Each line is a JSON object:

```typescript
// Relevant record types:

// Tool invocation (assistant turn)
{
  type: "assistant";
  message: {
    role: "assistant";
    content: Array<{
      type: "tool_use";
      id: string;          // toolId (e.g. "toolu_01XxYy...")
      name: string;        // Tool name (e.g. "Bash", "Write", "Read")
      input: {
        command?: string;  // For Bash
        file_path?: string; // For Read/Write/Edit
        description?: string; // For Task
        [key: string]: unknown;
      };
    } | { type: "thinking"; thinking: string }>;
  };
}

// Tool result (user turn)
{
  type: "user";
  message: {
    role: "user";
    content: Array<{
      type: "tool_result";
      tool_use_id: string;   // Matches the tool_use.id from assistant turn
      content: string | Array<{ type: "text"; text: string }>;
    } | { type: "text"; text: string }>;  // Text content = new user prompt
  };
}

// Turn-end signal (most reliable idle detection)
{
  type: "system";
  subtype: "turn_duration";
  durationMs: number;
}

// Subagent progress (Task tool forwarding inner messages)
{
  type: "progress";
  toolUseId: string;    // Parent Task tool_use.id
  data: {
    type: "agent_progress";
    message: {          // Inner assistant/user message from subagent
      role: "assistant" | "user";
      content: Array<tool_use | tool_result | text>;
    };
  };
}

// Long-running Bash output (restarts permission timer)
{
  type: "progress";
  toolUseId: string;
  data: {
    type: "bash_progress";
    output: string;
  };
}

// MCP tool status (same timer logic as bash_progress)
{
  type: "progress";
  toolUseId: string;
  data: {
    type: "mcp_progress";
    [key: string]: unknown;
  };
}
```

**Project hash formula:** Replace `:`, `\`, `/` with `-` in the workspace root path.
Example: `C:/Users/user/Projects/MyApp` → `C--Users-user-Projects-MyApp`
