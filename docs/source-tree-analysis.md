```
├── webview/
│   ├── index.html        ← Vite-generated HTML with script tags
│   └── assets/
│       └── index-[hash].js ← React + game engine bundle (ES module)

    ├── floors.png
    ├── walls.png
    ├── furniture-catalog.json
    └── default-layout.json
```

**Asset discovery in extension:** `assetLoader.ts` checks `dist/assets/` first (bundled), then falls back to workspace root (development).

### Extension Activation Entry Points

| File | Export | Purpose |
|---|---|---|
| `src/extension.ts` | `activate(context)` | Called by VS Code when extension loads |
| `src/extension.ts` | `deactivate()` | Called on extension unload |
| `webview-ui/src/main.tsx` | — (side effect) | React mount on webview load |

### Key Integration Points

| From | To | Mechanism |
|---|---|---|
| Extension Host | Webview | `panel.webview.postMessage()` |
| Webview | Extension Host | `vscode.postMessage()` (via `acquireVsCodeApi()`) |
| Extension Host | Filesystem | `fs.watch`, `fs.watchFile`, `fs.readFile`, `fs.writeFile` |
| Extension Host | Claude CLI | `terminal.sendText()` (send commands to terminal) |
| Scripts (pipeline) | `webview-ui/public/assets/` | Write PNG files + `furniture-catalog.json` |
| Claude CLI | Filesystem | Writes JSONL to `~/.claude/projects/<hash>/<session-id>.jsonl` |
